== Changelog ==

= 3.05 =

*	Fixed: Thumbnails within lightbox were not showing up (https://wordpress.org/support/topic/thumbnails-inside-a-lightbox/).
*	Fixed: When the number of columns was being set explicitly, the grids were displaying information in a single column (https://wordpress.org/support/topic/initial-number-of-albums-displayed/).
*	Fixed: "More" button was not showing up for WordPress galleries.
*	Fixed: For certain installations, CSS was not being read correctly; corrected by escaping the data uri SVG used in a background image (https://wordpress.org/support/topic/latest-update-3-04-breaks-website/).

= 3.04 =

*	Added: Full support for lazy-loading.
*	HTML / CSS Change: Cleaned up a lot of the HTML markup, reducing payload.
*	Added: Ability to show "More" for WordPress galleries.
*	Changed: Flickr photos now pick URLs from the url parameter instead of constructing based on the Farm and Server.

= 3.03 =

*	Changed: Updated PHP support to 7.1; no new syntax yet.
*	Changed: Updated WP compatibility support to 6.4
*	Changed: Renamed "Modules" to "Platforms" and "Core.php" to "Base.php"
*	Removed: Options for transpiled code, since IE support was dropped in version 3.00.
*	Added: Hooks <code>photonic_custom_sort_albums</code> and <code>photonic_custom_sort_photos</code> for sorting output before printing.
*	Changed: Updated lightbox scripts to latest versions - Lightgallery, PhotoSwipe, Splide
*	Fixed: Sort options in the Wizard were displaying for all types of Flickr level 1 displays (multiple / album / gallery photos) - corrected to only display for multiple photos (https://wordpress.org/support/topic/flickr-photoset-sorting-only-descending/)
*	Fixed: SmugMug keyword settings for albums, when defined through the Wizard, were not working (https://wordpress.org/support/topic/smugmug-filter-on-keywords-not-working-for-me/)
*	Fixed: Header options in a shortcode were not being deconstructed via the Wizard
*	Added: Ability to show LightGallery controls on mobiles (https://wordpress.org/support/topic/close-button-in-lightgallery-on-mobile/)
